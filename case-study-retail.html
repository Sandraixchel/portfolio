Featured case study

Retail Feedback Sentiment and Clustering

A multilingual customer-voice analysis across 42,000+ OLIST e-commerce reviews. I combined Python-based text processing, clustering, and an executive-level Power BI dashboard to show why delivery surfaced as the number one driver of customer churn.

  • Power BI
  • Python
  • SQL
  • Azure Translate
Timeline: 3 weeks · Sep 2025 Dataset: 42k e-commerce reviews · 9GB raw size

Explore the live dashboard

Use the filters on the left to focus on a segment, shipping window, or translated keyword cluster. Best viewed in full-screen mode.

Approach

I ingested the raw OLIST table exports into SQL Server, standardized product hierarchies, and tokenized/translated Portuguese comments to English via Azure Cognitive Services. After cleaning, I pushed the enriched table set into Power BI with custom M queries to refresh automatically.

Highlights

  • Delivery complaints represented 61% of all negative sentiment regardless of region.
  • Feature engineering exposed three hidden clusters: shipping-status anxiety, damaged packaging, and proactive gratitude.
  • Recommendations prioritized SLA transparency and packaging redesign, saving the ops team 4+ hours weekly.

What I would improve next

Add automated labeling using a lightweight topic model so each negative cluster can be piped directly into Zendesk as a proactive ticket tag, further reducing manual triage.
